# ADR-004

## Title

Program Bootstrap Architecture

---

## Status

ACCEPTED

LOCKED

---

## Date

July 2026

---

## Context

The Student Portal originally displayed abbreviated
credential information because program metadata was not
available during rendering.

Presentation components required Program Names,
Available Assets and Upgrade information.

Multiple options were considered.

---

## Alternatives Considered

### Option 1

CredentialCard queries Firestore directly.

Rejected.

Reason

Presentation layer must not access Firestore.

---

### Option 2

DashboardService queries Programs collection.

Rejected.

Reason

Business logic would become duplicated across services.

---

### Option 3

ProgramService loads Programs individually.

Rejected.

Reason

Creates one request per credential.

Poor scalability.

---

### Option 4

Cloud Run returns Programs during entitlement resolution.

Accepted.

---

## Decision

Cloud Run shall return the complete Programs collection
together with

• Credentials

• Executive Entitlements

• User Entitlements

The portal bootstraps the Programs collection into

window.__AAIU_PROGRAMS__

ProgramService becomes the single authority for resolving
Program ViewModels.

Presentation components consume only ProgramService.

---

## Consequences

Positive

✓ Single network request

✓ No duplicate Firestore reads

✓ Immutable Program cache

✓ Faster rendering

✓ Scalable architecture

✓ Cleaner separation of responsibilities

✓ Consistent Program metadata

---

Negative

Program bootstrap slightly increases entitlement payload
size.

This is acceptable because Program Definitions are small
and rarely change.

---

## Governance Decisions

LOCKED

Program metadata originates only from the Programs
collection.

Cloud Run owns business aggregation.

ProgramService owns Program resolution.

Presentation owns rendering only.

Credential presentation standard

Line 1

Program Code

Line 2

Program Name

No presentation component shall query Firestore.

---

## Outcome

The Program Bootstrap Architecture becomes the permanent
foundation for credential rendering throughout the
Student & Executive Portal.

Future modules shall consume ProgramService rather than
direct program data.
