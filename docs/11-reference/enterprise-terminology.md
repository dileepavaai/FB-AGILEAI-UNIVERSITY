# Enterprise Terminology Standard

**Version:** 1.0.0

**Status:** ACTIVE

**Classification:** Enterprise Standard

**Last Updated:** July 2026

---

# Purpose

The Enterprise Terminology Standard establishes the approved vocabulary for all Agile AI University documentation, architecture, development, governance, APIs, enterprise services, and operational documentation.

It ensures that every platform, document, service, API, and implementation uses a common enterprise language.

Consistent terminology reduces ambiguity, improves communication, and strengthens enterprise governance.

---

# Objectives

This standard exists to:

- Standardize enterprise language
- Eliminate ambiguity
- Improve documentation quality
- Support architecture governance
- Improve developer communication
- Preserve institutional consistency
- Enable long-term maintainability

---

# Governance

This document is authoritative.

Every document within the Agile AI University ecosystem shall use the terminology defined in this standard.

New terminology shall be reviewed before adoption.

---

# Enterprise Naming Principles

Enterprise terminology shall be:

- Consistent
- Precise
- Business-oriented
- Technology independent
- Easy to understand
- Stable
- Reusable

---

# Approved Enterprise Terms

---

## Agile AI

✅ Use

```
Agile AI
```

❌ Do not use

```
Agile + AI

Agile & AI

AI + Agile

AI and Agile
```

---

## Programme

✅ Use

```
Programme
```

❌ Avoid

```
Course

Training

Session
```

A Programme may contain learning, assessments, credentials, and recognitions.

---

## Learner

✅ Use

```
Learner
```

❌ Avoid

```
Student
```

"Learner" is the preferred enterprise term.

Student may be used only when referring specifically to the Student Portal.

---

## Enterprise Platform

✅ Use

```
Enterprise Platform
```

❌ Avoid

```
Application

App

Website

System
```

A Platform may contain multiple systems.

---

## Enterprise Service

✅ Use

```
Enterprise Service
```

❌ Avoid

```
Business Service

Backend Service
```

Enterprise Services expose Domain capabilities.

---

## Enterprise Domain

✅ Use

```
Enterprise Domain
```

❌ Avoid

```
Module

Component
```

Domains own business behaviour.

---

## Enterprise Registry

✅ Use

```
Enterprise Registry
```

❌ Avoid

```
Database

Table

Storage
```

The Registry represents institutional truth.

Implementation details remain independent.

---

## Credential

✅ Use

```
Credential
```

❌ Avoid

```
Certificate
```

A Certificate represents a Credential.

It is not the Credential itself.

---

## Certificate

Use only when referring to the visual representation of a credential.

---

## Digital Badge

Use only for the digital badge representation of a credential.

---

## Recognition

Recognition is not a Credential.

Recognition represents acknowledgement of achievement.

---

## Verification

Verification confirms authenticity.

Verification does not create credentials.

---

## Registration

Registration represents learner intent.

Registration is not payment.

---

## Payment

Payment represents financial settlement.

Payment does not create registration.

---

## Assessment

Assessment measures capability.

Assessment does not guarantee credential issuance.

---

## Learning

Learning represents the educational experience.

Learning is distinct from assessment.

---

## Executive Insight

Executive Insight represents enterprise analytics and decision support.

It is not a reporting dashboard.

---

# Architecture Terminology

- Platform Architecture
- Domain Architecture

- Integration Architecture

# Service Naming

Enterprise Services shall follow:

```
<ServiceName>Service
```

Examples:

```
ProgramService

RegistrationService

PaymentService

LearningService

AssessmentService

CredentialService

RecognitionService

VerificationService
```

---

# API Naming

Enterprise APIs shall follow:

```
<ServiceName> API
```

Examples:

```
Program API

Payment API

Credential API
```

---

# Registry Naming

Enterprise Registries shall follow:

```
<BusinessCapability> Registry
```

Examples:

```
Programme Registry

Registration Registry

Payment Registry

Credential Registry
```

---
